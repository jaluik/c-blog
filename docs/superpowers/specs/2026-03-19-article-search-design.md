## 1. 需求概述

为博客系统增加全文搜索功能，支持用户通过关键词搜索文章标题、摘要和正文内容。

### 核心需求
- 搜索字段：标题 + 摘要 + 正文
- 支持中文模糊匹配（搜索"前端"能匹配"全栈前端开发"）
- 关键词高亮显示
- 结果按相似度+时间排序
- 交互方式：Cmd+K 唤起搜索弹窗，支持快捷预览和完整搜索页

---

## 2. 技术选型

### 数据库搜索方案：PostgreSQL + pg_trgm

**选择理由：**
- 无需部署额外服务（vs MeiliSearch/Algolia）
- 天然支持中文模糊匹配（trigram 算法）
- 支持相似度计算和排序
- 100 篇文章数据量性能完全足够
- 与现有技术栈（Prisma + PostgreSQL）无缝集成

**替代方案考虑：**
- ~~PostgreSQL 全文搜索~~：中文分词需要额外配置，不够灵活
- ~~简单 LIKE 查询~~：无法计算相似度，排序不智能
- ~~独立搜索服务~~：运维成本高，小题大做

### 3.2 数据库设计

**扩展安装：**
```sql
-- 启用 pg_trgm 扩展
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- 创建搜索索引（合并标题、摘要、正文）
CREATE INDEX idx_articles_search ON articles
USING GIN ((title || ' ' || COALESCE(summary, '') || ' ' || content) gin_trgm_ops);
```

### 4.2 交互流程

**快捷搜索（弹窗模式）：**
1. 用户按 `Cmd+K` 或点击搜索图标
2. 弹出居中搜索框（毛玻璃效果，符合现有主题）
3. 输入关键词，防抖 200ms 后自动搜索
4. 显示前 5 条结果，高亮匹配关键词
5. 每条结果显示：标题 + 发布日期 + 匹配摘要片段
6. 底部显示"查看全部 N 条结果"链接

**完整搜索（结果页）：**
1. 从弹窗点击"查看更多"跳转 `/search?q=关键词`
2. 页面显示完整搜索结果列表
3. 支持分页加载
4. 空结果时显示友好提示

### 4.4 高亮实现

- 前端使用正则匹配关键词，包裹 `<mark class="search-highlight">` 标签
- 摘要截断：优先展示包含关键词的句子片段，前后各保留 50 字符

## 5. 排序算法

```typescript
// 综合评分公式
const finalScore = (similarity * 0.7) + (timeScore * 0.3);

// similarity: pg_trgm 返回的相似度 (0-1)
// timeScore: 基于发布时间的分数 (0-1，最新为 1)
```

## 6. 错误处理

### 6.1 空结果状态
- 显示友好提示："没有找到与 'xxx' 相关的文章"
- 提供建议：检查拼写、尝试其他关键词

### 6.2 网络错误
- 搜索失败时显示重试按钮
- 保留用户输入，方便重新搜索

### 6.3 输入限制
- 最少 1 个字符才触发搜索
- 最长 100 字符限制
- 自动去除首尾空格

---

## 7. 性能考虑

- **防抖**：输入后 200ms 才触发搜索，减少请求
- **取消请求**：新输入时取消未完成的搜索请求
- **索引**：GIN 索引确保查询性能 < 50ms
- **限制结果**：弹窗只显示前 5 条，结果页分页

## 附录：参考代码

### pg_trgm 查询示例
```sql
SELECT
  id, title, slug, summary, published_at,
  (title || ' ' || COALESCE(summary, '') || ' ' || content) <-> '搜索词' AS similarity
FROM articles
WHERE status = 'published'
  AND (title || ' ' || COALESCE(summary, '') || ' ' || content) % '搜索词'
ORDER BY similarity ASC, published_at DESC
LIMIT 10;
```
